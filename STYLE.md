GENERAL NAMING AND STYLE/FORMATTING RULES:

NAMING CONVENTIONS:

- Variables and functions use lowerCamelCase
- Classes use UpperCamelCase
- Constants use UPPER_SNAKE_CASE
- Names should be descriptive enough so others can understand what it is

CODE STYLE / FORMATTING:

- 2 spaces per indentation level
- No lines of code that are too long to ensure readability
- Code is clearly commented out if needed, explaining why something is done or what it does if the code is complex
- TODO comments should be left when needed with appropriate context
- Tests clearly describe which method they're testing

For any specific concerns on styling, formatting, etc., consult the google java style guide linked below:

https://google.github.io/styleguide/javaguide.html
